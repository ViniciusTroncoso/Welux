# Onboarding — Sócios Welux

Bem-vindo. Esse doc é pra você entrar no projeto em **15 minutos** e saber onde mexer.

---

## 1. Clone o repo

```bash
cd ~/Desktop
git clone https://github.com/ViniciusTroncoso/Welux.git
cd Welux
```

## 2. Visão de 30 segundos

- **Nicho:** médicos (de início — depois outros profissionais liberais premium).
- **Promessa:** autoridade digital. Paciente chega decidido, sem barganhar preço.
- **Modelo:** projeto único de criação de site (sem mensalidade obrigatória).
- **Stack principal:** Webflow + Relume para landing/sites, Hostinger para domínio/email.

## 3. Quem faz o quê

| Sócio | Papel principal | Responsabilidades |

| Vini (founder) | Estratégia + comercial | Posicionamento, copy, fechamento de clientes |
| [Sócio 2] | Design / Webflow | Implementação visual das páginas |

## 4. Como contribuir no repo

**Para mudanças pequenas (typos, ajustes de texto):**
- Crie um branch curto: `git checkout -b ajuste/copy-hero`
- Commit e abra PR
- Qualquer outro sócio aprova e dá merge

**Para mudanças grandes (mudança de posicionamento, estrutura nova):**
- Abra uma **issue** primeiro descrevendo a proposta
- Alinha com os sócios
- Depois faz o PR

**Não dê push direto na `main` sem PR**, mesmo sendo founder. Isso mantém histórico claro.

## 5. Convenção de commits

Curto e direto:
```
add: <coisa nova>
update: <coisa que já existia>
fix: <bug ou correção>
docs: <só mudança em .md>
brand: <mudança de identidade visual>
```

Exemplo: `docs: ajusta promessa do hero no posicionamento`

## 7. Princípios de trabalho

- **Decisão > debate.** Se tem dúvida, alguém decide e segue. Volta atrás se errar.
- **Padrão alto.** Welux é premium. Nada de entrega meia-boca pra cliente.
- **Cliente paga rápido, entrega rápida.** Ciclo de venda curto, projeto bem escopado.
- **Tudo documentado.** O que não tá no repo, não existe.
